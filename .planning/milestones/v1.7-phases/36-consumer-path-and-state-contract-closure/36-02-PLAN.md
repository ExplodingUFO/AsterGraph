# Plan 02: History Save Dirty Contract

## Objective

Publish the actual history/save/dirty rules as an explicit host-facing contract.

## Scope

- new or expanded consumer-facing documentation section for state semantics
- links from README and host docs

## Tasks

1. Distill the rules already proven by focused tests into a short contract.
2. Document:
   - what save does to dirty state
   - what undo does after save
   - what redo does when returning to the saved state
   - what no-op interactions do
   - that mixed retained/runtime flows still use one kernel-owned history authority
3. Point the docs at the proof lane and smoke markers that enforce the contract.

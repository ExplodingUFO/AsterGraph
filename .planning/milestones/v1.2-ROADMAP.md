# Milestone v1.2: Kernel Extraction, Capability Contracts, and Plugin Readiness

**Status:** ✅ SHIPPED 2026-04-08
**Phases:** 13-18
**Total Plans:** 18

## Overview

AsterGraph moved from a `GraphEditorViewModel`-centered runtime story to a kernel-first editor architecture with explicit capability/command/menu descriptors, thinner Avalonia adapters, and a proof ring that makes migration and plugin/automation readiness machine-checkable.

## Phases

### Phase 13: Editor Kernel State Owner Extraction

**Goal**: Extract the editor's canonical mutable state owner out of `GraphEditorViewModel`.
**Depends on**: v1.1 completed baseline
**Plans**: 3 plans

Plans:

- [x] 13-01: Introduce `GraphEditorKernel` as the first non-Avalonia runtime state owner
- [x] 13-02: Make `AsterGraphEditorFactory.CreateSession(...)` compose the kernel directly
- [x] 13-03: Lock kernel-first ownership into tests, `HostSample`, and `PackageSmoke`

**Details:**
Phase 13 established the kernel as the canonical runtime state owner, moved session composition off the `GraphEditorViewModel` facade for the canonical path, and closed with proof markers that made the new ownership model explicit.

### Phase 14: Session And Compatibility Facade Decoupling

**Goal**: Turn `IGraphEditorSession` into a kernel-facing API and reduce `GraphEditorViewModel` to an adapter/compatibility facade.
**Depends on**: Phase 13
**Plans**: 3 plans

Plans:

- [x] 14-01: Make retained `GraphEditorViewModel.Session` adapter-backed over the kernel
- [x] 14-02: Tighten the canonical query boundary around DTO/snapshot reads
- [x] 14-03: Lock retained-vs-canonical parity into proof/sample output

**Details:**
Phase 14 detached runtime session ownership from the retained facade, reframed MVVM-shaped query surfaces as compatibility shims, and proved that the retained session path now rides the shared runtime boundary instead of owning state itself.

### Phase 15: Capability And Descriptor Contract Normalization

**Goal**: Replace MVVM-shaped public control-plane seams with explicit capability, command, and menu descriptors.
**Depends on**: Phase 14
**Plans**: 3 plans

Plans:

- [x] 15-01: Add explicit runtime feature-descriptor discovery
- [x] 15-02: Normalize canonical command and menu contracts around stable descriptors
- [x] 15-03: Lock the descriptor-first story into proof/tests/docs/sample output

**Details:**
Phase 15 made capability discovery explicit, introduced stable runtime command/menu descriptor snapshots and command execution by ID, and moved retained menu rendering onto compatibility adapters over the canonical descriptor surface.

### Phase 16: Avalonia Adapter Boundary Cleanup

**Goal**: Make the Avalonia layer a thinner adapter over shared kernel/facade contracts.
**Depends on**: Phase 15
**Plans**: 3 plans

Plans:

- [x] 16-01: Share default shortcut and stock menu routing over session descriptors

- [x] 16-03: Lock the thinner adapter boundary into proof/tests/docs/sample output

**Details:**
Phase 16 removed duplicated shell/canvas routing policy, pushed stock context menus through canonical descriptor presenters, and made platform seam ownership explicit between the full shell and standalone canvas through a shared binder.

### Phase 17: Compatibility Lock And Migration Proof

**Goal**: Prove that the new kernel-first path and the retained facade path stay aligned during migration.
**Depends on**: Phase 16

Plans:

**Details:**
Phase 17 made the canonical runtime route, canonical hosted-UI route, and retained compatibility window explicit and consistently proven across tests, API remarks, quick-start docs, host integration docs, and runnable proof outputs.

### Phase 18: Plugin And Automation Readiness Proof Ring

**Goal**: Validate that the extracted kernel and capability contracts are strong enough to support later plugin and automation work.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Extract state ownership before building plugin or automation features.
- Keep `GraphEditorViewModel` and `GraphEditorView` as retained compatibility adapters rather than alternate runtime owners.
- Normalize capability, command, and menu discovery around descriptors and snapshots instead of MVVM object shape.
- Keep Avalonia as an adapter layer over shared runtime contracts rather than a policy source.
- Use focused tests plus `HostSample`, `PackageSmoke`, and `ScaleSmoke` as the proof ring for architectural claims.

**Issues Resolved:**

- Canonical runtime/session composition no longer requires constructing `GraphEditorViewModel`.
- Public control-plane discovery no longer depends on `RelayCommand`, `ObservableCollection`, or other MVVM implementation details as the canonical model.
- Shell and canvas no longer duplicate most shipped shortcut/menu/platform policy.
- Migration and readiness claims are now explicit and machine-checkable instead of doc-only assertions.

**Issues Deferred:**

- Actual runtime plugin loading, discovery, isolation, and trust model
- Rich automation or macro APIs over the extracted kernel
- Dedicated diagnostics workbench UI on top of the public probe surface
- The known retained history/save semantic mismatch (`STATE_HISTORY_OK`) that predates this milestone

**Technical Debt Incurred:**

- Compatibility shims remain public during the staged migration window.
- Copy/paste still executes through a compatibility bridge until canonical clipboard command IDs exist.
- Existing nullable warnings in kernel/adapter code paths remain to be cleaned up in later work.

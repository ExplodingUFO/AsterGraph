# Plan 43-02: Repair prerelease workflow evaluation

## Goal

Make `.github/workflows/release.yml` evaluate successfully on GitHub Actions without weakening the secret gate around NuGet publication.

## Steps

1. Remove direct `secrets.*` usage from job-level `if:` expressions.
2. Keep the release workflow tag/manual only, but move the NuGet-secret gate inside the publish job so the workflow can still evaluate without the secret.
3. Preserve verification-first behavior: validation jobs must finish before prerelease creation or NuGet push.

## Verification

- `gh workflow view .github/workflows/release.yml --yaml`
- `pwsh -NoProfile -ExecutionPolicy Bypass -File .\eng\ci.ps1 -Lane release -Framework all -Configuration Release`

<objective>
Define the canonical plugin-load inspection contract so hosts can read loaded-plugin descriptors, contribution shape, and failures through stable runtime queries and inspection snapshots instead of diagnostics text scraping.
</objective>

<scope>
- Add a stable public plugin inspection DTO surface in `AsterGraph.Editor`.
- Refine the internal loader result so it preserves per-registration reports needed for inspection.
- Expose the same plugin-load state through `IGraphEditorQueries` and `GraphEditorInspectionSnapshot`.
</scope>

<action>
- Introduce a new public plugin-load inspection DTO surface in `AsterGraph.Editor.Plugins`, covering at minimum registration source, resolved plugin descriptor when available, success/failure outcome, and a compact contribution summary.
- Refine the internal plugin loader result so it preserves one stable report per registration/load attempt instead of only aggregate successful descriptors plus diagnostics.
- Extend `IGraphEditorQueries` with a canonical plugin inspection read such as `GetPluginLoadSnapshots()`, and make `GraphEditorInspectionSnapshot` carry the same data.
- Keep `plugin.load.succeeded` / `plugin.load.failed` diagnostics intact, but make them supporting evidence rather than the only inspection path.
- Add an explicit discoverability signal such as `query.plugin-load-snapshots` so hosts can intentionally detect the new inspection query.
- Add focused contract/inspection tests that fail if plugin inspection regresses to diagnostics-only or if `Create(...)` / `CreateSession(...)` no longer expose equivalent plugin load state.
</action>

<acceptance_criteria>
- hosts can read stable plugin load snapshots from the canonical runtime query surface without scraping diagnostics text
- `GraphEditorInspectionSnapshot` includes the same plugin load state exposed by the new canonical query
- plugin inspection data includes both successful loads and recoverable failures
- the new inspection surface preserves parity across retained and runtime composition routes
- discoverability includes an explicit marker for plugin load inspection availability
</acceptance_criteria>

# 458.1 Editor Command And Undo Seam Audit

## Scope

This audit covers the Editor/runtime command plane only: command descriptors, command execution routing, command IDs, disabled and recovery state, undo/redo boundaries, and host extension seams. It does not cover Avalonia input routing or workbench chrome.

## Current Command Ownership

- `src/AsterGraph.Editor/Runtime/GraphEditorCommandDescriptorSnapshot.cs` defines the runtime command descriptor contract: stable `Id`, `Title`, `Group`, `IconKey`, `DefaultShortcut`, `Source`, `IsEnabled`, `DisabledReason`, `RecoveryHint`, and `RecoveryCommandId`.
- `src/AsterGraph.Editor/Runtime/Internal/GraphEditorCommandDescriptorCatalog.cs` owns stock descriptor metadata for canonical IDs such as `history.undo`, `history.redo`, `clipboard.copy`, `clipboard.paste`, `layout.align-left`, `viewport.fit`, and `workspace.save`.
- `src/AsterGraph.Editor/Kernel/Internal/CommandRouting/GraphEditorKernelCommandRouter.cs` owns stock enabled state and dispatch for kernel commands. `GetCommandDescriptors()` builds stateful descriptors from document, selection, history, permission, viewport, fragment, export, and connection state. `TryExecuteCommand(...)` switches on stable command IDs and delegates to the router host.
- `src/AsterGraph.Editor/Kernel/Internal/CommandRouting/GraphEditorKernelCommandRouterHost.cs` adapts router calls back into `GraphEditorKernel` without exposing kernel internals to the router.
- `src/AsterGraph.Editor/Runtime/Commands/GraphEditorSessionCommands.cs` is the public session command facade; `TryExecuteCommand(...)` delegates to the session host, while named command methods remain the strongly typed host-facing API.
- `src/AsterGraph.Editor/Runtime/Queries/GraphEditorSessionQueries.cs` exposes `GetCommandDescriptors()` through `IGraphEditorQueries`.

## Execution Flow

Current canonical execution path:

1. A caller asks `session.Queries.GetCommandDescriptors()`.
2. `GraphEditorSessionQueries.GetCommandDescriptors()` asks the host for descriptors and includes plugin command descriptors.
3. UI, automation, menus, tools, or host code builds a `GraphEditorCommandInvocationSnapshot`.
4. `session.Commands.TryExecuteCommand(...)` delegates to the host.
5. `GraphEditorKernel.TryExecuteCommand(...)` delegates to `GraphEditorKernelCommandRouter.TryExecuteCommand(...)`.
6. The router parses named arguments, delegates to `IGraphEditorKernelCommandRouterHost`, and returns `true` only for recognized accepted commands.

Important evidence:

- `src/AsterGraph.Editor/Kernel/GraphEditorKernel.cs:1770` exposes `GetCommandDescriptors()`.
- `src/AsterGraph.Editor/Kernel/GraphEditorKernel.cs:1773` exposes `TryExecuteCommand(...)`.
- `src/AsterGraph.Editor/Runtime/Commands/GraphEditorSessionCommands.cs:848` exposes session-level `TryExecuteCommand(...)`.
- `tests/AsterGraph.Editor.Tests/GraphEditorKernelCommandRouterTests.cs` covers descriptor state, command dispatch, recovery hints, and invalid argument rejection.
- `tests/AsterGraph.Editor.Tests/GraphEditorSessionTests.cs:1478` compares retained and runtime command IDs so the two session modes stay aligned.

## Undo And Redo Boundary

- `GraphEditorKernel.Undo()` and `GraphEditorKernel.Redo()` delegate to the history coordinator at `src/AsterGraph.Editor/Kernel/GraphEditorKernel.cs:161` and `src/AsterGraph.Editor/Kernel/GraphEditorKernel.cs:164`.
- Command descriptors for `history.undo` and `history.redo` are owned by the command descriptor catalog and stateful router.
- `GraphEditorKernelCommandRouter` exposes undo/redo as normal command IDs, but actual history mutation stays in the kernel history coordinator.
- Tests exercise one-action undo behavior for semantic operations such as reconnect, drop-node-on-edge, layout plan apply, group resize, and node drag.

Phase 459 should keep this split: registry metadata can describe undo behavior, but undo/redo execution remains a command over the existing history coordinator. Do not introduce a second history stack.

## Disabled And Recovery State

- Descriptors already carry `DisabledReason`, `RecoveryHint`, and `RecoveryCommandId`.
- `GraphEditorKernelCommandRouterTests` includes high-friction recovery coverage, recovery command ID validity, and permission-disabled behavior.
- Menu and tool descriptor builders consume descriptor enabled state instead of inventing local availability rules for many stock items.

Phase 459 can formalize this into a registry contract, but should not replace the router's source-backed state with static metadata. Availability is currently dynamic and depends on active document, selection, pending connection, history, permissions, and host storage.

## Host Extension Seams

- `GraphEditorCommandSourceKind` differentiates kernel and host/plugin command origins.
- `GraphEditorSessionPluginCommands.cs` can append plugin command descriptors and execute plugin commands through `GraphEditorPluginCommandExecutionContext`.
- `GraphEditorToolProvider` and stock tool/menu builders already let host-provided tools project command invocations without owning command execution.
- `GraphEditorCommandDescriptorSnapshot` fallback metadata currently titleizes unknown IDs, which is useful for plugin commands but is not enough for a professional command registry.

Phase 459 should make host command registration explicit enough for conflict checks, grouping, and keybinding metadata while still routing execution through session boundaries.

## Phase 459 Seams

Recommended narrow write areas:

- Add a package-level command registry/query model near `Runtime` or `Hosting`, not inside Avalonia chrome.
- Extend descriptor metadata or introduce a registry snapshot that preserves current `GraphEditorCommandDescriptorSnapshot` semantics.
- Keep `GraphEditorKernelCommandRouter` as the stock command availability and dispatch owner.
- Add conflict detection around default shortcut metadata and host shortcut policy, with focused tests near command router/session contracts.
- Keep plugin and host commands additive; do not create a parallel runtime command executor.

## Risks

- `GraphEditorKernelCommandRouter.GetCommandDescriptors()` is large and already owns mixed metadata, availability, recovery, and dispatch knowledge. Expanding it directly for every Phase 459 feature risks god code.
- `GraphEditorCommandDescriptorCatalog` has fallback metadata for unknown IDs. A registry that silently falls back for stock commands could hide missing supported metadata.
- `IGraphEditorCommands` is broad. Adding more named methods for every new command would further enlarge the public facade; descriptor-first execution should carry more of Phase 459.
- Undo behavior is implicit today. Phase 459 should classify command undo behavior for discovery/documentation, but must not move undo semantics out of existing transaction/history paths.

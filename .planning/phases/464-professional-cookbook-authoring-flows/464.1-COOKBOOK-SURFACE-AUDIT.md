# Phase 464.1 Audit: Professional Cookbook Authoring Flows

## Goal

Phase 464 should turn the Avalonia Demo cookbook into a code-plus-demo surface for v0.77 authoring workflows. It should demonstrate the supported package contracts from Phases 459-463 without making the Demo a generator, workflow runtime, query language, or fallback layer.

## Current Evidence

- `src/AsterGraph.Demo/Cookbook/DemoCookbookCatalog.Recipes.cs` owns the recipe catalog. It already contains recipes for starter host, authoring, performance/viewport, groups/subgraphs, plugin trust, diagnostics, and review help.
- `src/AsterGraph.Demo/Cookbook/DemoCookbookWorkspaceProjection.cs` projects recipes into left navigation, graph anchors, code examples, documentation links, scenario points, interaction facets, proof markers, route clarity, support boundaries, and deferred gaps.
- `src/AsterGraph.Demo/ViewModels/MainWindowViewModel.Cookbook*.cs` owns search/filtering, detail-mode state, selected scenario projection, graph lines, coverage lines, and detail lines.
- `src/AsterGraph.Demo/Views/MainWindow.axaml` already has the cookbook shape requested earlier: left navigation, graph/editor content above, and recipe/code/detail content below.
- `tests/AsterGraph.Demo.Tests/DemoCookbookCatalogTests.cs`, `DemoCookbookWorkspaceProjectionTests.cs`, `DemoCookbookNavigationTests.cs`, `DemoCookbookNavigationPolishTests.cs`, `DemoCookbookInteractionStateTests.cs`, `DemoCookbookDetailReadabilityTests.cs`, and `DemoCookbookVisualBaselineTests.cs` verify metadata, anchors, navigation, selection, detail modes, and visual structure.

## v0.77 Contract Evidence Available

- Command registry and keybinding surface: `IGraphEditorQueries.GetCommandRegistry()`, `GraphEditorCommandRegistryEntrySnapshot`, and `tests/AsterGraph.Editor.Tests/GraphEditorCommandRegistryTests.cs`.
- Semantic editing and clipboard: `IGraphEditorCommands.TryDeleteSelectionAndReconnect`, `TryDetachSelectionFromConnections`, connection selection/delete/slice routes, and `tests/AsterGraph.Editor.Tests/GraphEditorDeleteReconnectDetachTests.cs`.
- Template palette and reusable presets: `IGraphEditorCommands.TryApplyFragmentTemplatePreset`, fragment template snapshots, and `tests/AsterGraph.Editor.Tests/GraphEditorSessionFragmentContractsTests.cs`.
- Selection transform and snap guides: `IGraphEditorQueries.GetSelectionTransformSnapshot`, `selection.transform.move`, snap guide projection, and `tests/AsterGraph.Editor.Tests/GraphEditorSelectionTransformContractsTests.cs`.
- Navigation/search/focus: `IGraphEditorQueries.SearchGraphItems`, `GetViewportBookmarks`, `TryFocusGraphItemSearchResult`, bookmark commands, and `tests/AsterGraph.Editor.Tests/GraphEditorGraphItemSearchProjectionTests.cs` plus `GraphEditorNavigationFocusWorkflowTests.cs`.

## Gaps

- The cookbook catalog does not yet have a dedicated v0.77 professional authoring route that ties command registry, semantic editing, templates, selection transform/snap, and navigation/search/bookmarks together as one code-plus-demo flow.
- Existing scenario/interaction enums are broad. They can express old graph/metadata/support facets, but they do not distinguish command, template, transform, and navigation workflows strongly enough for v0.77 proof.
- `DemoCookbookWorkspaceProjection` currently projects text lines only. It does not provide a compact workflow step model with command ids, code anchors, graph cue targets, and proof markers aligned per v0.77 flow.
- `MainWindow.axaml` has the requested left-navigation plus graph-above-content structure, but the bottom cookbook panel is still mostly line lists. It needs a clearer code/demo/proof split for professional authoring flows.
- Tests prove generic cookbook completeness, but not that all v0.77 flows are represented with code anchors, demo anchors, proof markers, and support-boundary text.

- Do not add generated runnable Demo code, macro execution, query language, marketplace behavior, fallback behavior, or compatibility layers.

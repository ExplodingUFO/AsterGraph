# Plan 43-01: Stabilize workflow cache pathing

## Goal

Make hosted-runner cache behavior deterministic by pinning the NuGet global-packages directory to a workspace-local path in both GitHub workflows.

## Steps

1. Add workflow-level `env.NUGET_PACKAGES` pointing at `${{ github.workspace }}/.nuget/packages` in both `.github/workflows/ci.yml` and `.github/workflows/release.yml`.
2. Add a lightweight directory-bootstrap step before `actions/setup-dotnet` so cache save/cleanup never sees a missing target directory.
3. Keep the existing lane structure and artifact uploads unchanged.

- `pwsh -NoProfile -ExecutionPolicy Bypass -File .\eng\ci.ps1 -Lane all -Framework all -Configuration Release`

# 461.1 Template And Preset Contract Audit

Bead: `avalonia-node-map-48w.4.1`

Scope: read-only audit for Phase 461 template palette and reusable authoring preset work. This document maps current contracts and names implementation boundaries for 461.2 and 461.3. No source code, test, project, or bead state was changed.

## Current Behavior Map

### Node Templates

- Contract type: `src/AsterGraph.Editor/Runtime/GraphEditorNodeTemplateSnapshot.cs`.
- Source of truth: active node catalog definitions, not generated code or serialized graph fragments.
- Projection path:
  - `GraphEditorSessionQueries.GetNodeTemplateSnapshots()` reads `_descriptorSupport.Definitions`.
  - Each definition is projected through `GraphEditorNodeTemplateSnapshot.Create(...)`.
  - Results are sorted by `Category`, then `Title`, using ordinal string comparison.
- Snapshot fields include `DefinitionId`, catalog-derived `Key`, display text, size, accent, and input/output counts.
- Activation path today is still node insertion: `IGraphEditorCommands.AddNode(NodeDefinitionId, GraphPoint?)`, command id `nodes.add`, and the Avalonia stencil invokes that command through the hosted action/command route.
- Existing validation:
  - `GraphEditorSessionParameterContractsTests.RuntimeContracts_ExposeDefinitionAndParameterEditingSurface`
  - `GraphEditorSessionParameterContractsTests.SessionQueries_ExposeSharedDefinitionMetadata_AndParameterSnapshots`
  - `GraphContextMenuBuilderTests.EditorNodeTemplates_ExposeCanonicalTemplateSnapshots`
  - `GraphEditorViewTests.StencilLibrary_UsesSessionStencilItems_WhenRetainedTemplatesAreCleared`

### Edge Templates

- Contract type: `src/AsterGraph.Editor/Runtime/GraphEditorEdgeTemplateSnapshot.cs`.
- Source of truth: current active graph plus compatibility service, not stored reusable presets.
- Projection path:
  - `IGraphEditorQueries.GetEdgeTemplateSnapshots(sourceNodeId, sourcePortId)`
  - `GraphEditorSessionQueries.GetEdgeTemplateSnapshots(...)`
  - `GraphEditorKernel.GetEdgeTemplateSnapshots(...)`
  - `GraphEditorKernelCompatibilityQueries.GetEdgeTemplateSnapshots(...)`
- The compatibility query resolves the source output port, enumerates compatible input ports and compatible parameter targets, and returns host-safe snapshots with `Target`, endpoint kind, source/target types, accent, default label, and compatibility result.
- Edge templates currently describe valid target choices for an existing source port. They do not represent reusable authoring presets.
- Existing validation:
  - `GraphEditorSessionParameterContractsTests.RuntimeContracts_ExposeDefinitionAndParameterEditingSurface`
  - `GraphEditorSessionTests.GraphEditorSession_GetEdgeTemplateSnapshots_IncludeParameterEndpointsAndCreationDefaults`
  - `GraphContextMenuBuilderTests.BuildPortMenu_UsesCanonicalEdgeTemplates_ForParameterTargets`

### Fragment Templates

- Contract type: `src/AsterGraph.Editor/Runtime/GraphEditorFragmentTemplateSnapshot.cs`.
- Storage state type: `src/AsterGraph.Editor/Runtime/GraphEditorFragmentStorageSnapshot.cs`.
- Source of truth: persisted `GraphSelectionFragment` payloads in the fragment template library service.
- Projection path:
  - `IGraphEditorQueries.GetFragmentStorageSnapshot()`
  - `IGraphEditorQueries.GetFragmentTemplateSnapshots()`
  - `GraphEditorSessionQueries` delegates both to the session host.
  - `GraphEditorKernel.GetFragmentStorageSnapshot()` and `GetFragmentTemplateSnapshots()` delegate to `GraphEditorKernelFragmentStorageCoordinator`.
  - The coordinator maps `IGraphFragmentLibraryService.EnumerateTemplates()` into `GraphEditorFragmentTemplateSnapshot` and orders by `LastModified` descending, then `Name` ordinal.
- Storage service:
  - `IGraphFragmentLibraryService` exposes `LibraryPath`, `EnumerateTemplates()`, `SaveTemplate(...)`, `LoadTemplate(...)`, and `DeleteTemplate(...)`.
  - `GraphFragmentLibraryService` stores templates as JSON using the clipboard payload serializer, sanitizes file names, enumerates top-level `*.json` files, and computes node/connection counts by deserializing the fragment.
- Import/export command path:
  - `TryExportSelectionAsTemplate(string? name)` creates a selection fragment and saves it to the library.
  - `TryImportFragmentTemplate(string path)` loads a saved template and pastes it through the same fragment paste machinery as clipboard/workspace import.
  - `TryDeleteFragmentTemplate(string path)` deletes the template file.
- Existing validation:
  - `GraphEditorSessionFragmentContractsTests.RuntimeContracts_ExposeFragmentStorageAndTemplateLibrarySurface`
  - `GraphEditorSessionFragmentContractsTests.SessionQueries_ExposeFragmentStorageAndTemplateMetadata`
  - `GraphEditorSessionFragmentContractsTests.SessionCommands_RoundTripFragmentWorkspaceAndTemplateLibrary`
  - `GraphEditorViewTests.FragmentLibrary_UsesSessionSnapshots_WhenRetainedTemplatesAreCleared`

### Fragment Payloads From Phase 460

- Fragment creation copies only the induced subgraph of the current selection.
- Fragment payload includes selected nodes, selected-internal connections, origin, primary node id, and groups fully contained by the selected node set.
- Paste/import remaps:
  - node ids through `CreateNodeId(...)`
  - connection ids through `CreateConnectionId()`
  - group ids through collision-aware group id generation
  - node `Surface.GroupId`
  - group membership node ids
  - group positions by paste offset
- Paste/import is a graph mutation with `GraphEditorDocumentChangeKind.FragmentPasted`, publishes changed node/connection ids, and selects the pasted nodes.

### Stencil And Palette Projection

- Avalonia stencil controls live in `src/AsterGraph.Avalonia/Controls/GraphEditorView.axaml` and `GraphEditorView.axaml.cs`.
- State and controls:
  - `_stencilTemplateSnapshots`
  - `_stencilSearchBox`
  - `_stencilSourceFilter`
  - `_collapsedStencilCategories`
  - `_favoriteStencilTemplateKeys`
  - `_recentStencilTemplateKeys`
  - `_stencilCardList`
- Refresh path:
  - `RefreshCommandSurfaceState()` queries `Editor.Session.Queries.GetNodeTemplateSnapshots()` and stores the result in `_stencilTemplateSnapshots`.
  - `BuildStencilLibrary(...)` filters those cached snapshots, adds Favorites and Recent sections, groups remaining items by category, and applies `CurrentWorkbenchPerformancePolicy.StencilCardsPerSectionLimit`.
- Current source filter is inferred by template key today:
  - built-in vs plugin is based on `template.Key.Contains("provider", StringComparison.OrdinalIgnoreCase)`.
  - This is a UI heuristic, not a runtime contract.
- Current search is UI-local and matches snapshot text fields; it is deterministic for a fixed template list but not exposed as a reusable projection object.
- Fragment library UI is separate from stencil:
  - `BuildFragmentLibrary(...)` queries `GetFragmentStorageSnapshot()` and `GetFragmentTemplateSnapshots()`.
  - The picker stores `_selectedFragmentTemplatePath`.
  - Import/delete actions are rendered by `BuildFragmentTemplateActionToolbar()`.

### Template Import And Export Commands

- Public runtime commands:
  - `IGraphEditorCommands.TryExportSelectionAsTemplate(string? name)`
  - `IGraphEditorCommands.TryImportFragmentTemplate(string path)`
  - `IGraphEditorCommands.TryDeleteFragmentTemplate(string path)`
- Session command events publish:
  - `fragments.export-template`
  - `fragments.import-template`
  - `fragments.delete-template`
- Kernel command router currently routes `fragments.export-template`. Direct `TryImportFragmentTemplate` and `TryDeleteFragmentTemplate` exist on the public command surface, while stock Avalonia fragment template buttons call those methods through hosted actions rather than a general command invocation id.
- Template export is not an undoable document mutation because it writes the template library. Template import is undoable as a fragment paste document mutation through the kernel history path.

## Contract Decisions For Phase 461

- Templates and presets are source-backed graph fragments or catalog projections. They must not be treated as executable code generation.
- Node templates remain definition/catalog projections that insert one node by `NodeDefinitionId`.
- Edge templates remain compatibility projections for completing or authoring an edge from a live source endpoint.
- Reusable presets should build on `GraphSelectionFragment` payloads when they create reusable multi-node or configured-node authoring content.
- A reusable preset application should mutate the active graph through existing fragment paste or equivalent kernel mutation paths so it has one clear undo boundary.
- Host-provided templates must remain data contracts: hosts can provide definitions, fragment library services, and persisted fragments, but application still resolves through editor commands and compatibility rules.

Recommended implementation boundary:

- Keep node template source as `GetNodeTemplateSnapshots()`.
- Add a stable palette projection only if the UI needs reusable host-facing search/filter state. Otherwise keep the change inside Avalonia and document that search/filter remains a view concern.
- Replace the current source-filter heuristic with explicit metadata only if the runtime can prove it from catalog/provider data. If not, keep source filtering scoped to UI affordance and do not expose it as contract truth.
- Preserve ordering determinism:
  - normalized search text
  - ordinal comparisons
  - stable sort by source/category/title/key
  - performance limits applied after deterministic grouping
- Do not mix fragment templates into the node stencil unless the new contract explicitly creates a combined template/preset palette. If combined, use a new discriminated projection snapshot rather than overloading `GraphEditorNodeTemplateSnapshot`.

- Preserve template export/delete as library side effects, not undoable document changes.
- Keep import/preset application capability checks aligned with existing fragment import checks: template management, fragment import, node creation, and connection creation when the payload contains connections.

## Risks And Invariants

- Group ids: pasted group ids must avoid collisions with existing active-scope groups and with other pasted groups.
- Group membership: only groups fully contained by the selected node set should be exported; partial groups must not leak invalid membership.
- Node `Surface.GroupId`: imported nodes must reference the remapped group id or clear stale group ids when the source group is not included.
- Fragment payloads: Phase 460 payload shape is the contract for reusable presets; changing it affects clipboard, workspace fragments, and template library round trips.
- Undo boundaries: preset application must commit through one kernel mutation and one history entry.
- Host-provided templates: custom `IGraphFragmentLibraryService` implementations may enumerate templates in any order; runtime/UI projections should sort deterministically.
- Search/filter determinism: normalize user input and use ordinal comparisons so filtered results are stable across cultures.
- Public API exposure: do not expose Avalonia-only palette state from `AsterGraph.Editor` unless hosts need the same projection outside the stock Avalonia view.
- Edge templates: keep live compatibility queries separate from reusable presets. A preset should not bypass endpoint compatibility checks when it creates connections.
- File paths: fragment template commands accept paths. Keep validation around missing, invalid, or host-owned paths explicit and recoverable.
- Library side effects: export/delete modify disk state and should not be represented as document undo operations.

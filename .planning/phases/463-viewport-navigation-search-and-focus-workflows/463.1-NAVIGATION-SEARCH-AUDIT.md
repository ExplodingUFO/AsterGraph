# Phase 463.1 Audit: Navigation Search And Focus Workflows

## Goal

Phase 463 should expose source-backed graph search and navigation workflows for large graphs: stable search results, jump/focus commands, bookmarks, breadcrumbs, and minimap-aware focus. The implementation should reuse the existing viewport, scope, validation, command, and outline spine.

## Current Evidence

- Viewport commands already exist in `GraphEditorKernel`: pan, zoom, reset, fit scene, fit selection, focus selection, focus current scope, center node, and center world point.
- `GraphEditorCommandDescriptorCatalog` already declares viewport commands such as `viewport.fit`, `viewport.fit-selection`, `viewport.focus-selection`, `viewport.focus-current-scope`, `viewport.center-node`, and `viewport.center`.
- `GraphEditorSessionQueries.GetViewportSnapshot`, `GetSceneSnapshot`, `GetNavigatorOutlineSnapshot`, `GetScopeNavigationSnapshot`, `GetValidationSnapshot`, and `CaptureInspectionSnapshot` expose the state needed for navigation and focus.
- `GraphEditorNavigatorOutlineProjector.Project` is already called from session queries and should remain the source for outline-like navigation references.
- Validation issues are projected with stable scope/node/connection/parameter fields; Avalonia problems rows call `Editor.Session.Commands.TryFocusValidationIssue(issue)`.
- Scope breadcrumbs already have a package-level scope navigation snapshot and an Avalonia projection through `AsterGraphCompositeWorkflowActionFactory.CreateBreadcrumbActions`.
- The Avalonia surface has a minimap, validation panel, command palette search, stencil search, scope breadcrumbs, and focus restoration plumbing in `GraphEditorView.axaml.cs`.

## Gaps

- There is no unified graph item search query returning nodes, groups, connections, validation issues, and scopes as one stable snapshot.
- Existing command palette and stencil search are UI-specific filters; they should not become the graph item search contract.
- Bookmark state is not represented in Editor runtime snapshots or commands.
- Jump/focus workflows are split across node centering, validation issue focus, breadcrumbs, and minimap center; Phase 463 needs a single supported command/query surface.
- Large-graph claims need measured proof, not descriptive UI assertions.

## Recommended Bead Boundaries

### 463.2: Source-backed graph item search projection

Write scope:
- `src/AsterGraph.Editor/Runtime/IGraphEditorQueries.cs`
- `src/AsterGraph.Editor/Runtime/Queries/GraphEditorSessionQueries.cs`
- New runtime snapshots for graph item search query, result kind, result target, and result snapshot.
- Tests under `tests/AsterGraph.Editor.Tests/*NavigationSearch*`.

Expected work:
- Search active and full document graph items by stable fields: id, title, definition/category, group title, connection endpoints/labels, validation issue code/message, and scope id.
- Return stable result kinds and target references for node, group, connection, issue, and scope.
- Include deterministic ordering and limit behavior.
- Add a measured large-graph test that proves projection stays bounded and deterministic.

### 463.3: Bookmark breadcrumb and focus command workflows

Write scope:
- `src/AsterGraph.Editor/Runtime/IGraphEditorCommands.cs`
- `src/AsterGraph.Editor/Runtime/Commands/GraphEditorSessionCommands.cs`
- `src/AsterGraph.Editor/Kernel/Internal/CommandRouting/*`
- New runtime bookmark/focus snapshots if needed.
- Minimal Avalonia consumption in `src/AsterGraph.Avalonia/Controls/GraphEditorView*`.
- Focused tests for command routing, bookmark state, breadcrumbs, and focus outcomes.

Expected work:
- Add command routes for jump-to-node, jump-to-issue, jump-to-search-result, add/remove/activate viewport bookmark, and focus current scope/selection where missing.
- Keep breadcrumbs backed by `GraphEditorScopeNavigationSnapshot`; do not duplicate scope state in UI.
- Keep minimap-aware focus as a command over viewport snapshots/world points, not a second viewport model.
- Make command availability/recovery hints come from the same descriptor path as earlier phases.

## Constraints

- Do not add an indexing service, background search service, query language, or compatibility fallback.
- Do not treat command palette/stencil search as graph item search.
- Do not move viewport state into Avalonia.
- Keep supported contracts in Editor package snapshots/commands; Avalonia only projects and invokes them.

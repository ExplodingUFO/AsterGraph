# Phase 04 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Sources: `04-CONTEXT.md`, `04-RESEARCH.md`, approved `03-UI-SPEC.md`, `AsterGraphTheme.axaml`, `GraphEditorView.axaml`, `GraphInspectorView.axaml`, `NodeCanvas.axaml.cs`, `GraphMiniMap.cs`, and `GraphContextMenuPresenter.cs`.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - Avalonia-only manual contract sourced from `04-CONTEXT.md`, `04-RESEARCH.md`, the existing `AsterGraphTheme`, and current stock presenter surfaces. Phase 03 spacing and typography defaults carry forward where Phase 04 does not explicitly override them. |
| Preset | not applicable |
| Component library | Avalonia built-in controls plus AsterGraph public stock surfaces and per-surface presenter seams in `AsterGraph.Avalonia`: `GraphEditorView`, `NodeCanvas`, `GraphInspectorView`, `GraphMiniMap`, `GraphContextMenuPresenter`, and optional replacement presenters for node visuals, menus, inspector, and mini map |
| Icon library | none - stock menus use built-in text glyph chips; replacement presenters may add host visuals but every action must keep a text label |
| Font | Inter for demo and stock-shell validation; packable library surfaces inherit the host font unless the host pins a font explicitly |

## Visual Hierarchy

| Surface | Primary Focal Point | Secondary Focal Point | Accessibility Rule |
|---------|---------------------|-----------------------|--------------------|
| Node presenter | Node title and active port topology | Subtitle, top-right badges, and status bar | Interactable ports keep adjacent text labels and left/right direction cues; custom chrome cannot rely on color alone to communicate port meaning |
| Context-menu presenter | Action label and group hierarchy | Icon chip, submenu affordance, and disabled reason | Every command remains text-labeled; separators never carry meaning alone, and disabled items surface their reason through tooltip or inline hint |
| Inspector presenter | Current selection title, category, and parameter editors | Connection summaries, upstream/downstream lists, and validation text | Editable fields keep visible labels plus mixed-value and validation messaging; no icon-only field affordances |
| Mini-map presenter | Current viewport rectangle | Node extents and selected-node highlight | The surface stays non-focusable and pointer-driven; no hidden-only controls or unlabeled affordances |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Micro gaps between captions, badge text, and dense validation copy |
| sm | 8px | Menu row rhythm, compact port rows, and tight inspector grouping |
| md | 16px | Default presenter padding, parameter card spacing, and menu content inset |
| lg | 24px | Standalone surface padding blocks and node card section separation |
| xl | 32px | Host layout gaps between independently replaced surfaces |
| 2xl | 48px | Empty-state breathing room and major presenter section breaks |
| 3xl | 64px | Outer host padding around full-shell or showcase compositions |

Exceptions: Existing stock theme resources still contain compatibility values such as `10px`, `12px`, `14px`, `18px`, and `22px`; new Phase 4 presenter APIs, examples, and docs should expose only the declared 4-point tokens and treat legacy values as stock-implementation internals.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09111A / #0C1825 / #0A121C` | Shell backdrop, stock presenter demo background, and full-shell convenience composition |
| Secondary (30%) | `#111C29` | Node shells, inspector shells, mini-map container chrome, stock context-menu body, and fallback presenter surfaces |
| Accent (10%) | `#7FE7D7` | Selected node borders and header wash, compatible or pending connection emphasis, inspector category emphasis, mini-map viewport frame, and stock node badges only |
| Destructive | `#FF9B9B` | Validation errors, destructive labels, and permanent fragment/template deletion affordances only |

Accent reserved for: selected node borders and header wash; compatible or pending connection emphasis; selected-node badges and status bars; inspector category and current-selection emphasis; mini-map viewport framing. Do not use accent as the default menu hover, panel background, or generic button fill.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Add Node |
| Empty state heading | Select A Node |
| Empty state body | Build the graph from the left library, connect outputs to inputs, and save snapshots from the toolbar. |
| Error state | Action failed. Retry the command. If a custom presenter cannot render, keep the stock presenter visible and surface the editor session status message in host chrome. |
| Destructive confirmation | `Delete Selected`, `Delete Node`, `Delete Connection`, `Disconnect`, and `Break Connections`: no blocking confirmation in stock presenters; use explicit destructive labels and immediate status feedback because the actions stay within the live session. `Clear Fragment`: `Clear the saved fragment snapshot for this host?` `Delete Template`: `Delete the selected fragment template from this host library?` |

## Presenter Replacement Contract

| Surface | Contract |
|---------|----------|
| Node visual replacement | `NodeCanvas` remains the owner of selection, drag, marquee, pan/zoom, pending connection preview, guide rendering, and anchor resolution. Custom node presenters may replace the node body, header, badges, and status visuals, but they must render category/title/subtitle, description, separate input and output port regions, and hit-testable port anchors wired back to editor-owned node and port state. |
| Context-menu replacement | `GraphEditorViewModel.BuildContextMenu(...)` and `MenuItemDescriptor` remain the source of truth. Replacement presenters may change popup chrome, nesting visuals, and icon treatment, but they must preserve command wiring, enabled/disabled state, separator structure, child menus, disabled reasons, and stock `GraphContextMenuPresenter` fallback when no custom presenter is provided. |
| Inspector replacement | Replacement presenters must bind to the existing `GraphEditorViewModel` inspector projections and `NodeParameterViewModel` editing flow. They must keep `InspectorTitle`, `InspectorCategory`, `SelectionCaption`, `InspectorDescription`, `Connections`, `Inputs`, `Outputs`, `Upstream`, `Downstream`, `Parameters`, mixed-value hints, and validation text inside one surface instead of inventing a parallel inspector state model. |
| Mini-map replacement | Replacement presenters may change drawing primitives and surrounding chrome, but they must stay narrow: graph extents, selected-node emphasis, current viewport frame, and click or drag recentering through editor-owned viewport APIs. No status bar, menu host, or toolbar responsibilities get folded into the mini-map seam. |

## Interaction Contract

| Surface | Contract |
|---------|----------|
| Shared session binding | All presenter surfaces bind to the same `GraphEditorViewModel`; custom Avalonia presenters may reshape visuals, but they do not duplicate node, menu, inspector, or viewport state. `INodePresentationProvider` and `IGraphContextMenuAugmentor` remain upstream enrichment seams and must continue to feed replacement presenters. |
| Opt-in replacement | Presenter replacement is per-surface and opt-in. `GraphEditorView` remains the convenience full shell, standalone surfaces remain directly embeddable, and omitted or null custom presenters always resolve to the stock default for that surface. |
| Node canvas behavior | Canvas behaviors remain unchanged: node drag, selection, marquee selection, port activation, connection preview and creation, wheel zoom, panning, alignment guides, and right-click targeting. Replacement node visuals cannot require hosts to reimplement those behaviors or bypass `NodeCanvas`. |
| Menu baseline | The stock reference menu keeps background `#111C29`, border `#22354A`, hover `#193045`, foreground `#F4FBFF`, disabled foreground `#6A8091`, `220px` min width, `12x8` item padding, `10px` menu radius, `8px` item radius, and `18px` glyph chips. Custom menu presenters may diverge visually, but stock remains the documented fallback baseline. |
| Focus and input ownership | Canvas shortcuts only fire while the canvas owns focus. Text and combo editors inside inspector presenters suppress graph shortcuts while editing. Menu close returns focus to the canvas or originating control. Mini-map presenters remain non-focusable. |
| Fallback and failure | If a replacement presenter is unavailable or throws during composition, the host path falls back to the stock presenter for that surface and preserves editor interaction. Recoverable presentation failures surface through existing status text or host chrome instead of disabling the editor session. |

## Architecture Decisions

1. Keep `NodeCanvasInteractionSession` as the single transient-mode source of truth for canvas gestures.
2. Keep pure pointer routing, resize hit testing, viewport math, and visible-scene projection in `AsterGraph.Editor` where they can be tested without Avalonia controls.
3. Keep Avalonia pointer capture, focus, cursor feedback, visual control lifecycles, and menu presentation in `AsterGraph.Avalonia`.
4. Keep viewport state canonical in the editor/session model. Do not introduce Avalonia-owned viewport state.
5. Keep command, menu, hosted action, keyboard, and context behavior on the canonical session command route.
6. Preserve the keyboard split: `GraphEditorView` owns hosted shell shortcuts and command-palette focus; `NodeCanvas` owns canvas descriptor shortcuts and pending connection cancel.
7. Do not plan fallback layers, compatibility shims, renderer rewrites, or a second interaction runtime for v0.78.

## Cross-Cutting Risks

- Release-time gesture commits are centralized in `NodeCanvasPointerInteractionCoordinator.HandleReleased`. Future work should split responsibilities carefully, preserving release vs capture-loss semantics.
- Resize start paths are intentionally split across explicit handles and edge hit testing. Production changes must update stock visuals, scene host, resolver, and pure hit-test proof together.
- Visible-scene projection exists but is not yet used to bound actual Avalonia visual work. Phase 467 should prove measured behavior before making performance claims.
- Connection rendering still clears and recreates visual paths. Any bounding change must keep pending connection preview and resize-preview anchors live.
- Minimap throughput mode is a policy contract. Freshness and deferral must be tested and documented explicitly.
- Custom node/body/editor controls can disrupt shortcut routing if they do not participate in input-scope suppression.
- Context menu customization should not bypass `IGraphEditorCommands.TryExecuteCommand`; otherwise disabled state, events, and plugin/kernel routing diverge.

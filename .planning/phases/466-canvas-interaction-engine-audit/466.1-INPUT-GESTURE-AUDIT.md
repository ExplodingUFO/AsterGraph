## Gesture Lifecycle Summary

1. Node drag: node visual pointer press enters `DefaultGraphNodeVisualPresenter` or a custom visual callback, `NodeCanvas.BeginNodeDrag` delegates to `NodeCanvasNodeDragCoordinator`, session stores drag origin, `NodeCanvas` captures the pointer, move applies drag assist and `ApplyDragOffset`, release applies group membership if needed, completes history, resets session, clears capture.
2. Group drag: group header pointer press enters `NodeCanvasSceneHost`, `BeginGroupDrag` resolves move mode and selected member nodes, session stores frame origin and member origins, move previews frame and/or offsets members, release commits group frame position when needed, completes history, resets session, clears capture.
3. Node resize: node thumb or border edge press enters `BeginNodeResize`, session stores node resize origin and preview, resize feedback cursor is locked, move updates preview and rerenders node/connections without persisting, release commits `TrySetNodeSize`, completes history, resets session, clears capture.
4. Group resize: group thumb or border edge press enters `BeginGroupResize`, session stores frame origin and preview, move applies edge math plus assist and rerenders groups/connections, release commits `TrySetNodeGroupFrame`, completes history, resets session, clears capture.
5. Canvas selection: background left press enters `NodeCanvasPointerInteractionCoordinator.HandlePressed`; the router begins canvas selection and may cancel pending connection preview. Move crosses threshold to become marquee selection; overlay converts screen rectangle to world and updates selection with Ctrl/Shift modifiers; release finalizes selection or clears selection if it never became marquee.
6. Panning: middle press or enabled Alt+left press begins panning, captures pointer, move calls `PanBy` with screen delta, release resets.
7. Wheel: wheel does not use pointer capture. The wheel coordinator updates last pointer position; Control/Meta wheel calls `ZoomAt`, otherwise wheel calls `PanBy`.
8. Capture loss: `PointerCaptureLostEvent` calls coordinator cleanup only when a transient mode is active; cleanup resets session and clears guide/resize feedback, but does not commit partial drag/resize mutations.

## Risks And Coupling Points

- `NodeCanvasPointerInteractionCoordinator.HandleReleased` is a high-coupling commit hub: node drag group-membership changes, group position, node resize, group resize, marquee finalization, history completion, pending connection cancellation, and feedback cleanup all share one method.
- Capture cleanup is intentionally non-committing. Future phases that add cancel/commit distinction must preserve the current difference between release and capture loss.
- Node resize start is split between explicit resize thumbs and border-edge hit-testing. Both must continue to call `NodeCanvas.BeginNodeResize` so custom visual presenters and stock visuals keep the same lifecycle.
- Group resize start is split between explicit thumbs and root/header border-edge hit-testing. Future edge/handle changes must update `NodeCanvasSceneHost`, `NodeCanvasResizeFeedbackResolver`, and `GraphEditorSceneResizeHitTester` together.
- Marquee selection depends on `SelectionBaselineNodes`, `SelectionModifiers`, visible hierarchy filtering, and `GetNodesInRectangle`; hidden/collapsed group behavior can regress if selection work only tests raw node bounds.
- Drag assist is shared by node drag, group drag, and group resize. Small snapping changes can alter all three gesture families.
- Resize preview state lives in `NodeCanvasInteractionSession` while rendered visuals read previews through `NodeCanvasSceneHost.ResolveRendered*` helpers. Persisted model state must not be updated during move unless the intended behavior changes.
- Wheel gesture policy is controlled by `EnableDefaultWheelViewportGestures` and is host-cooperative. Any new zoom/pan owner should keep the disable path intact.
- `GraphEditorPointerInputRouter` is pure and tested, but Avalonia-specific pointer properties and handled-events behavior still live in `NodeCanvas` and scene/presenter code.
- Pointer capture is owned by `NodeCanvas`, not coordinators. Coordinators return capture intent, but start paths for node/group resize capture directly. A future lifecycle refactor should normalize this without changing behavior.

## Missing Proof Points

- No focused test proves pointer capture is released after every direct resize start path and after every drag path in the same parameterized matrix.
- Capture-loss tests cover reset behavior, but not whether active resize cursor/session feedback is restored for every node and group resize mode.
- Wheel tests cover disabled/pan/zoom routing, but not high-resolution or horizontal-only wheel deltas through the integrated `NodeCanvas` event path.
- Coordinator tests cover a top-edge group resize preview, but integrated standalone tests emphasize right/bottom edges more than all four group edges.
- Custom node visual presenters are tested for callbacks and anchors, but there is no narrow proof that a custom visual's node resize callback remains behavior-identical to stock resize thumbs.
- Marquee tests cover collapsed-group visibility, but not Ctrl-toggle and Shift-add modifier behavior through the integrated canvas event path.
- Release/capture-loss history behavior is not exhaustively separated for resize sessions; capture loss should stay non-committing and non-history-completing.

## Handoff Notes For Integration

- Treat `NodeCanvasInteractionSession` as the single transient-mode source of truth. New gestures should add explicit state there or prove they can remain stateless.
- Keep pure routing/hit-test code in `AsterGraph.Editor/Scene` when the logic does not need Avalonia types; this keeps low-level behavior easy to test.
- Do not move persisted mutations into move handlers unless a phase explicitly changes preview semantics. Current drag/resize move paths are preview-oriented except node drag position offsets, which are wrapped by history interaction.
- Preserve host switches: `EnableDefaultWheelViewportGestures`, `EnableAltLeftDragPanning`, `CommandShortcutPolicy`, and `ResizeFeedbackPolicy`.
- Integration synthesis should watch for overlap with command/platform audits: pointer gestures call session commands and view-model facade helpers, but this audit only owns gesture lifecycle and canvas interaction boundaries.

# Code of Conduct

## Our Standard

This project expects a respectful, technically rigorous collaboration style.

Examples of behavior that contribute to a healthy environment:

- discussing ideas and code directly, without personal attacks
- accepting good-faith technical disagreement
- giving actionable, evidence-based review feedback
- reporting bugs, regressions, and security concerns responsibly
- leaving room for contributors with different levels of experience

Examples of unacceptable behavior:

- harassment, insults, or demeaning language
- threats, intimidation, or sustained hostility
- publishing private information without permission
- trolling, deliberate disruption, or bad-faith review behavior
- sexualized language or imagery in project spaces

## Scope

This code of conduct applies to repository issues, pull requests, discussions, and other project-managed communication channels.

## Enforcement

Maintainers may remove, edit, or reject comments, commits, issues, pull requests, and other contributions that violate this code of conduct.

## Reporting

For behavior issues, contact the maintainers through the repository's private security/contact channel when possible. For product security issues, follow [`SECURITY.md`](./SECURITY.md).
